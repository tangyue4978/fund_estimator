# Fund Estimator 系统梳理

## 系统定位

Fund Estimator 是一个基于 Streamlit 的基金组合估值与日结分析系统，面向个人基金组合的盘中估值、持仓回放、收盘日结、官方净值覆盖和误差复盘。

## 功能地图

- 登录与会话：`services/auth_guard.py`、`services/auth_service.py` 负责注册、登录、Cookie 会话和本地开发用户。
- 自选基金：首页 `app/Home.py` 展示自选基金实时估值，支持添加、移除、跳转详情。
- 持仓管理：`app/pages/01_持仓.py` 基于流水回放生成指定日期持仓，支持手工编辑、清仓、删除流水和图片识别导入。
- 日结台账：`app/pages/02_日结.py` 生成收盘估算，尝试用官方净值覆盖，支持扫描近 7 天待结算记录。
- 基金详情：`app/pages/03_基金详情.py` 展示单基金实时估值、官方净值走势、实时估值走势、个人收益走势和误差历史。
- 组合分析：`app/pages/04_组合分析.py` 提供组合曲线、收益归因、目标仓位偏离和数据健康检查。
- 数据源适配：`datasources/` 封装基金档案、实时估值、官方净值、HTTP 缓存等外部数据访问。
- 业务服务层：`services/` 承载估值、持仓、流水、日结、误差、图表、图片识别、组合分析等业务逻辑。
- 本地存储：`storage/` 按用户隔离 JSON 文件，并提供带锁的原子读写。

## 核心数据流

1. 用户登录后，`storage.paths.current_user_id()` 决定当前用户数据目录。
2. 自选列表提供基金代码集合，估值服务批量拉取实时估值。
3. 持仓以流水为主口径，通过 `snapshot_service` 回放到指定日期。
4. 盘中页面使用实时估值计算组合市值、盈亏和覆盖率。
5. 日结页面把收盘估算写入台账，再用官方净值覆盖为 settled 状态。
6. 组合分析从日结、流水和目标仓位中生成曲线、归因和健康检查结果。

## 本次完善

- 首页自选列表增加组合概览指标：自选数量、平均涨跌幅、上涨/下跌数量、提示数。
- 首页自选列表增加排序能力：支持默认顺序、按涨跌幅、按置信度排序。
- 首页自选列表增加提示筛选：可只查看存在估值提示或风险提示的基金。
- 首页自选列表和基金详情走势使用“上涨红、下跌绿”的视觉口径。
- 日结台账增加顶部统计：台账记录、已覆盖官方净值、待覆盖估算、最近日期。
- 日结台账增加状态筛选和基金代码筛选，便于快速定位待处理记录。
- 日结台账的“删除超卖 SELL 流水”维护工具增加确认开关，避免误点执行破坏性操作。
- 组合分析增加收益归因概览：今日预计收益、正/负贡献数量、最大驱动标的和驱动收益。
- 组合分析增加目标仓位概览：目标仓位合计、最大偏离金额、需关注标的数量。
- 组合分析收益、偏离和曲线颜色统一使用上涨红、下跌绿。
- 数据检查增加错误、警告、提示、通过数量概览。
- 保持云端 Supabase 为主数据口径，不引入额外本地存储逻辑。

## 后续优化建议

- 基金详情页“实时估值走势”当前更接近实时点位展示；若需要真正盘中曲线，应接入稳定的盘中采样服务。
- 日结、持仓、组合分析的云端失败提示可以进一步归类为“配置缺失、网络失败、权限失败、表结构不匹配”，便于用户自助定位。
- 维护工具可继续增加操作预览，例如先列出将删除的异常流水，再执行删除。
- 为 `watchlist_service`、`portfolio_import_service`、`portfolio_analysis_service` 增加单元测试，覆盖云端失败、脏数据和重复数据场景。
- 对组合曲线增加资金流修正口径，避免大额申赎或调仓导致指数被组合规模变化扭曲。
- 图片导入可以增加“导入批次”记录，便于回滚某次识别导入。
- 健康检查页可以增加一键跳转到异常流水或异常日结记录，缩短修复路径。
